## Summary

### Key Principles

1. **Unidirectional Data Flow**: Data flows from backend â†’ frontend
2. **Event-Driven**: Changes trigger Socket.IO events
3. **Centralized Services**: All HTTP/WebSocket calls go through services
4. **Component Isolation**: Each page manages its own state
5. **Real-time Sync**: Database is source of truth, Socket.IO keeps UIs in sync

### Performance

- **Initial Load**: REST API fetch (one-time)
- **Updates**: WebSocket push (instant, no polling)
- **Database**: Prepared statements (cached queries)
- **Frontend**: React re-renders only affected components
